
# 🧒 왜 변수와 상수가 필요할까?

## 🎒 1. 변수(Variable)는 “필통” 같은 것

필통 안에 연필을 넣을 수도 있고, 색연필을 넣을 수도 있지?
그리고 언제든지 꺼내서 **바꿔 넣을 수도 있어**.

**변수도 똑같아!**

* 처음엔 `5`를 넣었다가
* 나중엔 `10`을 넣어도 돼
* 기존 건 사라지고 새 값이 들어옴

👉 **계속 바뀌는 값**을 저장할 때 변수 사용!

예)

* 오늘 기온은 -5도
* 내일은 4도
* 모레는 8도

날씨는 매일 변하니까 **변수로 저장하면 딱 맞아!**

---

## 🛑 2. 상수(Constant)는 “이름표” 같은 것

학생 이름표는 바뀌지 않지?
옷에 붙은 ‘김철수’라는 이름표는 하루종일 그대로잖아.

상수도 똑같아!
**처음 정한 값에서 절대 바뀌지 않음.**

👉 **절대로 변하면 안 되는 값**을 저장할 때 사용!

예)

* 원주율 3.14159 → 항상 일정
* 게임에서 목숨 최대값 3 → 바뀌면 안 됨
* 신호등의 빨강 = 멈춤 → 바뀌면 큰일남

---

# 🎯 왜 굳이 둘을 나눠? 그냥 다 변수로 하면 안 돼?

## 이유 1️⃣ 실수 방지

예를 들어,

* 너희 집 주소가 “서울시 OO구 OO동”인데
  누가 실수로 바꿔버리면 큰일 나겠지?

그래서 **절대로 바뀌면 안 되는 정보는 상수로 잠궈두는 것**이 좋아.

---

## 이유 2️⃣ 컴퓨터에게 “이건 변할 거야/안 변할 거야” 알려주기

컴퓨터는 네가 어떤 값을 쓸지 미리 알고 있으면
더 빠르고 안정적으로 계산할 수 있어.

* **변수는 바뀌는 값** → "아하, 나중에 또 바뀔 수 있구나!"
* **상수는 안 바뀌는 값** → "좋아! 이건 고정이니까 안전하게 저장하자!"

---

## 이유 3️⃣ 사람이 프로그램을 이해하기 쉬워짐

예를 들어,

```
const SPEED = 10;  // 상수 → 절대 바뀌지 않는 값
let currentSpeed = 10; // 변수 → 지금 속도, 나중에 바뀔 수 있음
```

이렇게 써놓으면 봤을 때 바로 이해됨!

---

# 🧃 한 줄 요약!

* **변수 = 바꿀 수 있는 값 → 필통**
* **상수 = 바꾸면 안 되는 값 → 이름표**

둘을 나누면
✔ 프로그램이 더 안전하고
✔ 이해하기 쉽고
✔ 컴퓨터도 더 빨리 계산할 수 있어!

선언을 **왜 꼭 해야 하는지**, 그리고 **선언과 비슷한 의미의 단어들**을 쉽게 정리해서 설명해줄게.

---

# 🧒 왜 “선언”을 해야 할까?

## 📌 1. **컴퓨터에게 ‘이름표 달아주기’**

변수나 상수는 **값을 넣어둘 상자** 같은데,
상자에는 **이름표**가 있어야 안 헷갈려.

예)

```
let age = 10;
```

여기서 `age`라는 이름표를 붙인 거야.

**선언 = 이름표 붙이기**

이걸 하지 않으면 컴퓨터가
“이 값이 어디 들어가는 거야?” 하고 모르게 됨.

---

## 📌 2. **컴퓨터에게 ‘앞으로 이런 걸 쓸 거야’ 알려주기**

컴퓨터는 사람처럼 추측하지 못해.
미리 알려줘야 준비를 함.

“나 지금 age라는 변수 만들 거니까 기억해!”
이게 바로 **선언**이야.

---

## 📌 3. **메모리를 준비해야 하기 때문**

컴퓨터 안에는 값을 저장하는 공간(메모리)이 있어.
변수나 상수를 선언하면 컴퓨터가 그 공간을 딱 준비함.

선언 없이 값을 넣으려고 하면
“어디다 넣어?” 하고 컴퓨터가 혼란스러워짐.

---

## 📌 4. **실수를 막기 위해**

선언 없이 막 값을 쓰면
다른 코드랑 섞여서 큰 오류가 생길 수 있어.

선언은 그걸 막는 **안전벨트** 같은 역할을 해.

---

# 🔤 ‘선언’과 비슷한 뜻의 단어들

프로그래밍에서 많이 나오는 단어들인데
뜻이 헷갈리기 쉬워서 정리해줄게!

| 단어                  | 쉬운 설명                      |
| ------------------- | -------------------------- |
| **선언(Declare)**     | “이 이름으로 변수를 만들게!” 하고 말하는 것 |
| **정의(Define)**      | “선언 + 값이나 내용도 만들어 넣기”      |
| **초기화(Initialize)** | 변수를 만들면서 첫 번째 값을 넣는 것      |
| **할당(Assign)**      | 이미 있는 변수에 값을 넣는 것          |
| **대입(Set)**         | 할당과 같음. 값을 넣는 행동           |
| **갱신(Update)**      | 기존 값이 있고 → 새 값으로 바꾸는 것     |
| **참조(Reference)**   | 변수 안에 있는 값을 읽어오는 것         |

---

# 🧃 예로 한 번 정리해볼게

```
let age;        // 선언(Declare)
age = 10;       // 할당/초기화(Initialize)
age = 20;       // 갱신(Update)
console.log(age); // 참조(Reference)
```

또는

```
let age = 10;   // 선언 + 초기화
```

---

# 🎯 한 줄 요약

* **선언은 컴퓨터에게 “이 이름으로 저장공간 만들 거야!” 하고 알려주는 과정**
* 선언을 해야 컴퓨터가 *어디에*, *어떤 이름으로*, *어떤 값을 넣을지* 알 수 있음
* 선언 = 이름표 붙이기 / 준비하기

# 🧒 변수와 상수 + 문자열 붙이기 쉬운 예제

---

## 📌 1) 변수 출력 + 문자열 붙이기

변수는 **바꿀 수 있는 값**이야.

```javascript
// 변수 만들기
let myAge = 10;

// 문자열 + 변수 붙여서 출력
console.log("내 나이는 " + myAge + "살 입니다.");

// 변수 값 바꾸기
myAge = 11;

// 다시 출력
console.log("바뀐 나이는 " + myAge + "살 입니다.");
```

출력

```
내 나이는 10살 입니다.
바뀐 나이는 11살 입니다.
```

---

## 📌 2) 상수 출력 + 문자열 붙이기

상수는 **바꾸면 안 되는 값**이야.

```javascript
// 상수 만들기
const schoolName = "푸른초등학교";

// 문자열 + 상수 붙이기
console.log("제가 다니는 학교는 " + schoolName + "입니다.");
```

출력

```
제가 다니는 학교는 푸른초등학교입니다.
```

---

## 📌 3) 변수 + 상수 함께 출력

문자열 붙여서 문장처럼 만들기!

```javascript
let studentName = "김철수";
const grade = 3;

console.log("이름: " + studentName);
console.log("학년: " + grade + "학년");
console.log(studentName + " 학생은 " + grade + "학년입니다.");
```

출력

```
이름: 김철수
학년: 3학년
김철수 학생은 3학년입니다.
```

---

## 📌 4) 계산 결과도 문자열과 함께 출력

```javascript
let a = 5;
let b = 3;

// 그냥 숫자 출력
console.log("a:", a);
console.log("b:", b);

// 문자열 + 계산 결과 붙이기
console.log("a와 b를 더하면 " + (a + b) + "입니다.");
console.log("a - b = " + (a - b));
console.log("a × b = " + (a * b));
```

출력

```
a: 5
b: 3
a와 b를 더하면 8입니다.
a - b = 2
a × b = 15
```

---

# 🎯 한 번에 이해 포인트

* `"문자" + 변수` → 둘을 이어 붙여서 한 문장처럼 만들 수 있음
* 변수(let) → 바뀌는 값
* 상수(const) → 바뀌면 안 되는 값
* console.log() → 화면에 보여줘!
