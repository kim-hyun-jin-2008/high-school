
# 🧠 1. 수도코드(글로 정리한 단계)

### 📌 “주차 요금 계산”을 인간이 이해하기 쉽게 단계로 풀어쓴 것

```
1. 주차한 시간을 저장한다.
2. 요금 저장용 변수를 0으로 준비한다.

3. 만약 주차 시간이 1시간 이하라면
    → 요금은 2000원으로 정한다.

4. 그렇지 않고 주차 시간이 3시간 이하라면
    → 요금은 5000원으로 정한다.

5. 위 조건 둘 다 아니라면
    → 요금은 10000원으로 정한다.

6. 마지막으로
    주차 시간과 요금을 화면에 출력한다.
```

이것이 **컴퓨터에 시키기 전, 사람이 먼저 생각하는 과정**이야.

---

# 💻 2. 실제 JavaScript 코드

```javascript
console.log("\n=== 예제 9: 주차 요금 계산 ===");

let parkingHours = 3;
let fee = 0;

if (parkingHours <= 1) {
    fee = 2000;
} else if (parkingHours <= 3) {
    fee = 5000;
} else {
    fee = 10000;
}

console.log("주차 시간: " + parkingHours + "시간");
console.log("주차 요금: " + fee + "원");
```

# 🔍 3. 논리적인 사고란 무엇인지, 왜 필요한지?

### ✔ 논리적인 사고란?

어떤 문제를 **순서 있게, 이유가 맞아 떨어지게** 생각하는 방법이야.

예를 들어,

* “시간이 1시간 이하이면 2000원이다.”
* “1시간보다 크지만 3시간 이하이면 5000원이다.”
* “그보다 많으면 10000원이다.”

이렇게 **조건을 하나씩 따져서 정확한 결론**을 만들어내는 것이 논리적 사고야.

---

### ✔ 왜 필요할까?

프로그래밍은 "컴퓨터에게 시키는 일"이라서
컴퓨터는 스스로 판단하지 못하고 **순서대로 명령을 읽기만** 해.

그래서 우리가,

* 어떤 상황에서 어떤 결과가 나와야 하는지
* 어떤 조건을 먼저 검사해야 하는지
* 어떤 상황이 맞지 않으면 다음 조건으로 넘겨야 하는지

이런 것들을 **논리적으로 정리해 줘야 해.**

---

### ✔ 논리적 사고가 없으면 어떻게 될까?

예를 들어, 다음 조건의 순서를 뒤바꾸면 문제가 생겨.

❌ `parkingHours <= 3` 를 가장 먼저 검사하면
1시간인 경우에도 잘못된 요금(5000원)이 나와버림.

즉, “순서”가 매우 중요해.

---

### ✔ 정리

* **수도코드** = 사람이 먼저 생각하는 순서
* **조건문** = 생각한 순서를 컴퓨터가 이해하도록 만드는 도구
* **논리적 사고** = 조건들을 정확한 순서로 배치하고 이유를 따지는 과정
* 이런 과정을 통해 컴퓨터가 **틀리지 않고 정확히 작동하도록** 할 수 있어
