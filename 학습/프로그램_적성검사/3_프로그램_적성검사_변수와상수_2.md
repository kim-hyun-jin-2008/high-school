# 📘 1. 리터럴(Literal) 정리 표

리터럴은 **값 그 자체를 표현하는 기호(표기)** 입니다.
즉 “이미 완성된 값”을 코드로 직접 적어 놓은 것.

| 종류        | 리터럴 예시                    | 의미          | 특징              |
| --------- | ------------------------- | ----------- | --------------- |
| 숫자 리터럴    | `10`, `3.14`              | 숫자 값        | 항상 동일한 값        |
| 문자열 리터럴   | `"hello"`, `'A'`, `"123"` | 텍스트 값       | 변경되지 않는 고정된 문자열 |
| 불린 리터럴    | `true`, `false`           | 참/거짓 값      | 두 개뿐            |
| null 리터럴  | `null`                    | 의도적으로 비어 있음 | 고정된 하나의 값       |
| undefined | `undefined`               | 값이 없음       | 특별한 값           |
| 배열 리터럴    | `[1, 2, 3]`               | 배열 값        | 내부 값이 고정 리터럴    |
| 객체 리터럴    | `{name: "Tom"}`           | 객체 값        | 구조 형태가 리터럴      |

### ✔ 핵심 정리

* `"123"` → 리터럴
* `123` → 리터럴
* `[1,2,3]` → 리터럴
* `{}` → 리터럴

**리터럴은 이미 완성된 “값 자체”이므로 바뀌지 않음.**

---

## 📌 `"123"` 같은 값도 상수인가?

아래는 `"123"` 같은 **문자열 리터럴이 왜 상수인지**,
그리고 **변수/상수에 저장되는 값과 리터럴 자체의 차이**까지 초보자도 이해하기 쉽게 설명한 내용입니다.

**그 자체로는 “상수(값이 변하지 않는 리터럴)”이지만,
변수에 저장하면 그 변수는 let/const에 따라 달라진다.**

조금 더 쉽게 말하면:

* `"123"` → **바뀌지 않는 고정된 값(리터럴)**
* `let a = "123"` → a는 **변할 수 있는 변수**
* `const b = "123"` → b는 **바뀌지 않는 상수**

즉, `"123"`이라는 **값 자체는 이미 완성된 고정 데이터**이기 때문에 바뀌지 않음.
하지만 **그 값을 담는 저장 공간(변수/상수)이 변경 가능하냐가 중요함**.

---

### 📘 리터럴(literal)이란?

프로그래밍에서 **리터럴**은 “그 자체로 값을 의미하는 표기”를 말합니다.

→ 이들은 모두 **값 자체가 변하지 않는 상수 형태**입니다.

→ 하지만 “저장하는 방식”에 따라 변수 또는 상수가 됨.



### 📌 정리

| 표현                | 의미          | 변경 가능?      |
| ----------------- | ----------- | ----------- |
| `"123"`           | 고정된 문자열 리터럴 | 자체적으로는 변경 X |
| `let a = "123"`   | 변수에 저장      | 값 변경 가능     |
| `const b = "123"` | 상수에 저장      | 값 변경 불가     |

# 📘 2. 변수 / 상수 / 리터럴 차이 심화 설명

아래는 세 개의 개념을 명확하게 구분해주는 핵심 요약입니다.

---

## ✔ 개념으로 구분하면

### 🔹 **리터럴**

* 값 자체(고정된 값)
* `"Hello"`, `10`, `true`, `null`, `{}`, `[]` 등이 모두 리터럴
* **값은 절대 변하지 않음**
* 다만 “그 값을 담는 곳(변수/상수)”에 따라 수정 가능 여부가 달라짐

---



### 🔹 **변수(let)**
* “값을 바꿀 수 있는 저장 공간”
* 프로그래밍에서 변수는 이름을 붙여서 값을 저장해 두는 상자 같은 것입니다.
* 필요할 때 꺼내 쓰고, 값을 나중에 바꿀 수 있습니다.

* 값을 담는 **저장 공간** -> 저장된 값을 **언제든 변경 가능**

```js
let a = "123";   // a는 변수
a = "456";       // 변경 가능
```

---

### 🔹 **상수(const)**
* “값을 바꿀 수 없는 저장 공간”
* 상수(const) 는 한 번 값을 넣으면 절대 변경할 수 없습니다.
* 중요한 값, 변하면 안 되는 값에 사용합니다.

* 값을 담는 저장 공간이지만 -> 한 번 넣으면 **변경 불가**

```js
const b = "123";
b = "456";  // ❌ 오류 — 바꿀 수 없음
```

---

## ✔ 핵심 비유로 이해하기

* **리터럴 = 물건(값)**
* **변수(let) = 뚜껑 열고 다시 넣을 수 있는 상자**
* **상수(const) = 한 번 넣으면 못 여는 상자**

물건은 바뀌지 않지만
상자(변수/상수)에 어떤 물건(리터럴)을 넣을지는 개발자가 결정.

---

## ✔ 프로그램에서 실제 동작 비교

```js
// ① 리터럴
"123"        // 값 자체
10           // 값 자체

// ② 변수에 저장
let x = "123";
x = "456";   // ✔ 변경 가능

// ③ 상수에 저장
const y = "123";
// y = "456";  // ❌ 오류! y는 변경 불가
```

# 📘 3. 세 가지를 하나의 표로 비교 정리

| 구분            | 의미       | 예시                    | 변경 가능 여부   | 특징             |
| ------------- | -------- | --------------------- | ---------- | -------------- |
| **리터럴**       | 값 그 자체   | `"123"`, `10`, `true` | 값 자체는 변경 X | 데이터 그 자체       |
| **변수(let)**   | 값을 담는 공간 | `let a = 10;`         | ✔ 값 변경 가능  | 가장 많이 사용       |
| **상수(const)** | 값을 담는 공간 | `const b = 10;`       | ❌ 값 변경 불가  | 변경되면 안 되는 값 저장 |

# 📘 4. 실습 예제 (Node.js 실행 가능)

## ✔ 리터럴 출력

```js
console.log("문자열 리터럴:", "123");
console.log("숫자 리터럴:", 456);
console.log("불린 리터럴:", true);
console.log("객체 리터럴:", {name: "Tom"});
console.log("배열 리터럴:", [1, 2, 3]);
```

---

## ✔ 변수와 상수의 차이 실습

```js
let v = "변경 가능";
const c = "변경 불가";

console.log("변수 v:", v);
console.log("상수 c:", c);

// 변수는 변경 가능
v = "값을 바꿨습니다!";
console.log("변경된 v:", v);

// c = "바꾸기";  // ❌ 오류 발생 (실행하면 에러 출력)
```

---

## ✔ 변수에 리터럴을 넣고 바꿔보기

```js
let num = 10;
console.log("num:", num);

num = 20;  // 리터럴 변경 X, 변수 값 변경 O
console.log("변경된 num:", num);
```

## 📘 흐름도를 해설로 정리해보면

### 1) **자료형이 먼저 존재한다**

* 어떤 값이든 먼저 자료형(숫자·문자·불린 등)이라는 분류가 있음.

### 2) **리터럴은 자료형을 가진 ‘값 그 자체’**

예:

* `"Hello"` → 문자열 리터럴
* `123` → 숫자 리터럴
* `{}` → 객체 리터럴
  리터럴은 **변하지 않는 값**.

### 3) **변수·상수는 값을 담는 ‘상자’**

* 변수(let)는 뚜껑 열고 다시 넣을 수 있음
* 상수(const)는 한 번 넣으면 잠김

### 4) **프로그램은 변수/상수에 저장된 값을 가져와 실행된다**

* 계산
* 출력
* 조건문 비교
* 반복문 사용
  모두 변수/상수 안의 값을 통해 이루어짐.
