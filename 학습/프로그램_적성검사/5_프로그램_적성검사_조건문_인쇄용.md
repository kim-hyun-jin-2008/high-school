# ⭐ 조건문이란?

조건문은 컴퓨터에게
**“만약 ~라면 이렇게 해!”**
**“그게 아니면 다른 걸 해!”**
라고 **상황에 따라 다른 선택을 하도록** 알려주는 도구예요.

사람이 말하는 것과 비슷해요.

* “비 오면 우산 가져가자!”
* “숙제 다 했으면 게임 할 수 있어!”
* “점수가 높으면 A등급!”

이런 말을 컴퓨터에게 알려주는 게 바로 **조건문**이에요.

---

# 📌 조건문 종류 표 (아주 쉽게)

| 이름                      | 실제 모양                                   | 하는 일                    | 언제 쓰는가       | 중요 포인트         |
| ----------------------- | --------------------------------------- | ----------------------- | ------------ | -------------- |
| **if**                  | `if (조건) { 실행 }`                        | 조건이 참일 때만 실행            | 가장 기본        | 혼자 써도 됨        |
| **if - else**           | `if (조건) { } else { }`                  | 조건이 참이면 if, 아니면 else 실행 | 두 가지 중 하나 선택 | else는 조건 없음    |
| **if - else if - else** | `if (조건) { } else if (조건) { } else { }` | 여러 가지 조건을 순서대로 검사       | 조건이 2개 이상일 때 | else는 맨 끝에 1번만 |

---

# ⭐ 왜 조건문을 사용해야 할까?

### ✔ 1. "상황에 따라 행동이 달라지기 때문"

사람도 상황에 따라 다르게 행동해요.
컴퓨터도 똑같이 필요해요.

예) 비 오면? → 우산
비 안 오면? → 우산 X
→ 상황이 다르니까 행동이 달라야 함 → 조건문 필요!

---

### ✔ 2. 조건문이 없으면 컴퓨터는 무조건 같은 행동만 함

조건문이 없다면…

예)

* 점수가 95점이어도 “A 등급”이라고 못 말함
* 나이가 5살이어도 “성인입니다.” 라고 말할 수 있음
* 아이디가 틀려도 계속 “로그인 성공”이라고 할 수도 있음 😱

즉, **조건문이 없으면 프로그램이 제대로 작동하지 않고, 말도 안 되는 결론이 나옴.**

---

### ✔ 3. 프로그램이 똑똑해지기 위한 필수 도구

조건문이 있으면 프로그램이:

* 상황 판단 가능
* 다른 결과 출력 가능
* 오류 방지 가능
* 더 다양한 기능 가능

조건문은 **뇌** 같은 역할이에요.
없으면 프로그램은 “생각 없는 로봇”처럼 행동해요.

---

# 📘 조건문 예제

```javascript
let age = 10;

if (age >= 18) {
  console.log("성인입니다.");
} else {
  console.log("어린이입니다.");
}
```

---

# 🔎 조건문이 없을 때의 문제를 쉽게 보기

### ❌ 조건문 없음:

```javascript
console.log("성인입니다."); // 무조건 출력됨
```

→ 1살이어도 “성인입니다.”라고 출력됨
→ 완전 잘못된 결과

---

### ✔ 조건문 사용:

```javascript
let age = 1;

if (age >= 18) {
  console.log("성인입니다.");
} else {
  console.log("어린이입니다.");
}
```

→ 나이에 따라 정확한 결과 출력

---

# ⭐ 마지막 요약

* **조건문은 “만약 ~라면!”이라고 컴퓨터에게 알려주는 문법**
* 조건에 따라 **다른 결과**를 만들 수 있음
* 없으면 프로그램이 **똑똑하게 판단하지 못하고**, 엉뚱한 값을 출력함
* 그래서 게임, 앱, 프로그램, 웹사이트 등 모든 곳에서 꼭 필요함
* 조건문은 컴퓨터의 **판단력**, **두뇌**, **선택 기능**이라고 보면 됨!
